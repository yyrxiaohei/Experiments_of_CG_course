# 1. 项目架构

Work3 相关项目目录结构如下：
```
CG_Lab/
├── pyproject.toml
├── src/
│   └── Work3/
│       ├── __init__.py    --> （包标识文件）
│       ├── config.py      --> 配置模块
│       ├── geometry.py    --> 曲线几何计算模块
│       ├── rasterizer.py  --> GPU光栅化渲染模块
│       └── main.py        --> 主程序入口模块
```


# 2. 代码逻辑
## （1）项目运行入口
在项目根目录（src上级目录）路径下，通过uv执行以下命令：
```
uv run -m src.Work3.main
```

## （2）关键模块说明
a. `config.py` **配置模块**
- 核心作用：统一管理贝塞尔曲线绘制实验中的全局参数。
- 分组定义：
  - 窗口系统参数：窗口标题、窗口宽度、窗口高度、窗口分辨率；
  - 曲线采样参数：曲线采样段数`NUM_SEGMENTS`、曲线采样点缓冲区大小`MAX_CURVE_POINTS`；
  - 控制点参数：最大控制点数量`MAX_CONTROL_POINTS`、贝塞尔曲线最少控制点数量、B样条曲线最少控制点数量；
  - 渲染颜色参数：背景颜色、曲线颜色、控制点颜色、控制多边形颜色；
  - 渲染样式参数：控制点半径、控制多边形线宽、反走样开关与反走样半径；
  - 对象池参数：控制点对象池容量、控制线段对象池容量、屏幕外隐藏坐标；
  - 曲线模式参数：贝塞尔曲线模式`MODE_BEZIER`、B样条曲线模式`MODE_BSPLINE`。
- 特点：所有参数集中定义，其他模块通过导入常量使用，便于统一调整实验效果。

b. `geometry.py` **曲线几何计算模块**
- 核心作用：在 CPU 端完成曲线点坐标的批量计算。
- 关键组成：
  - `de_casteljau()`：使用 De Casteljau 算法计算贝塞尔曲线在指定参数`t`处的坐标；
  - `generate_bezier_curve_points()`：对`t ∈ [0, 1]`进行均匀采样，批量生成贝塞尔曲线上的采样点；
  - `uniform_cubic_bspline_point()`：根据均匀三次 B 样条基函数计算单个局部曲线点；
  - `generate_uniform_cubic_bspline_points()`：遍历所有局部分段，生成完整的均匀三次 B 样条曲线采样点。
- 特点：该模块只负责数学计算，不直接参与窗口交互与 GPU 像素绘制。

c. `rasterizer.py` **GPU光栅化渲染模块**
- 核心作用：基于 Taichi Field 与 Kernel 实现像素缓冲区管理和曲线点光栅化。
- 关键组成：
  - 显存数据定义：创建`pixels`作为屏幕像素缓冲区，存储每个像素的 RGB 颜色；
  - 曲线点缓冲区：创建`curve_points_field`，用于接收 CPU 批量计算得到的曲线采样点；
  - 控制点对象池：创建`gui_points`，用于固定容量存储交互控制点；
  - 控制线对象池：创建`control_line_vertices`，用于固定容量存储控制多边形线段端点；
  - 清屏内核`clear_pixels()`：GPU 并行清空像素缓冲区，恢复背景色；
  - 光栅化内核`draw_curve_kernel()`：GPU 并行读取曲线采样点，将归一化坐标映射为像素坐标，并点亮对应像素；
  - 对象池重置内核`reset_gui_object_pools()`：将未使用对象移动至屏幕外隐藏位置。
- 特点：该模块承担主要 GPU 并行任务，是实现流畅绘制和避免 Python 逐点写显存卡顿的关键。

d. `main.py` **主程序入口模块**
- 核心作用：统筹程序执行流程，连接用户交互、曲线计算、GPU光栅化与窗口显示。
- 执行流程：
  - 初始化 Taichi GPU 环境；
  - 创建 GGUI 窗口与画布对象；
  - 监听鼠标左键点击事件，将鼠标位置记录为控制点；
  - 监听`C`键事件，清空全部控制点；
  - 监听`B`键事件，在贝塞尔曲线与均匀三次 B 样条曲线模式之间切换；
  - 每帧清空像素缓冲区；
  - 当控制点数量满足要求时，在 CPU 端批量计算曲线采样点；
  - 使用`from_numpy()`将曲线点一次性传入 GPU Field；
  - 调用 GPU Kernel 执行曲线光栅化；
  - 更新控制点对象池和控制多边形对象池；
  - 将`pixels`显示到窗口画布，并绘制控制点与控制多边形。

$\downarrow$

`config.py`提供统一参数 → `geometry.py`基于控制点计算曲线采样坐标 → `rasterizer.py`将采样点并行光栅化到像素缓冲区 → `main.py`调度交互、计算、渲染与窗口刷新，实现完整的交互式曲线绘制系统。


# 3. 实现功能

基于 Taichi 实现 GPU 加速的交互式贝塞尔曲线光栅化绘制系统，程序可响应鼠标点击动态添加控制点，并实时生成对应曲线，具体功能如下：
- 鼠标交互添加控制点：通过鼠标左键在窗口中添加控制点，并实时显示为红色圆点；
- 控制多边形绘制：按照控制点添加顺序，用灰色线段连接相邻控制点，直观展示贝塞尔曲线的几何控制结构；
- De Casteljau 曲线生成：基于递归线性插值算法，在 CPU 端批量计算贝塞尔曲线采样点；
- GPU 像素级光栅化：将曲线采样点批量传入 GPU Field，并通过 Taichi Kernel 并行点亮像素缓冲区中的对应位置；
- 实时曲线更新：每次新增控制点后，程序自动重新计算并绘制新的曲线形态；
- 键盘清空功能：按下`C`键后清空所有控制点，并恢复初始空白画面；
- 固定对象池绘制：预先分配控制点与控制线段 Field，避免主循环中动态申请显存；
- 批量 CPU-GPU 数据传输：CPU 端一次性计算所有采样点，再通过`from_numpy()`批量上传至 GPU，减少频繁跨设备通信；
- 反走样扩展支持：通过配置项`ENABLE_ANTI_ALIASING`控制是否启用基于 3x3 邻域距离衰减的平滑曲线绘制；
- B 样条曲线扩展支持：通过`B`键可在贝塞尔曲线模式与均匀三次 B 样条曲线模式之间切换，用于对比两类曲线的几何特性。
